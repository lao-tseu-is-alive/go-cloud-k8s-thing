This action plan is designed for an LLM agent to remediate technical debt and security risks in the `go-cloud-k8s-thing` repository. The plan prioritizes critical security vulnerabilities and "quick-win" operational fixes.

### Project Overview & Context
* **Stack**: Go 1.25, ConnectRPC, Vanguard (transcoding), PostgreSQL (PostGIS), Kubernetes.
* **Architecture**: Layered design with clean separation between Transport (`ConnectServer`), Business logic (`BusinessService`), and Persistence (`Storage`).
* **Deployment**: Containerized using multi-stage Docker builds (targeting `scratch`) and deployed via Kubernetes manifests.
* **Exclusion**: Do not address Kubernetes Secret vs. ConfigMap findings as they are already resolved.

---

### Remediation Action Plan

#### Phase 1: Critical Security & Data Integrity
1.  **Remove Credential Logging (Critical)**
    * **Task**: Locate `cmd/goCloudK8sThingServer/goCloudK8sThingServer.go` and delete lines 76 and 86.
    * **Reason**: These lines log password hashes at the `Debug` level, which is a major security anti-pattern.
2.  **Fix Soft-Delete Integrity (Critical)**
    * **Task**: Update the `Storage` layer to ensure queries for `Get`, `Update`, and `Delete` explicitly filter for `deleted_at IS NULL`.
    * **Reason**: Currently, "deleted" entities remain accessible or modifiable via direct ID access.
3.  **Restrict Over-Permissive RBAC (Critical)**
    * **Task**: Modify `deployments/go-testing/deployment.yml` to remove `ClusterRoleBinding` to `system:serviceaccounts`.
    * **Reason**: Granting pod-reader/service-reader roles cluster-wide to all service accounts is an excessive security risk.
4.  **Enforce API Pagination Limits (High/Quick-Win)**
    * **Task**: In `BusinessService`, add a hard cap (e.g., 500) to the `limit` parameter for all list/search operations.
    * **Reason**: Unbounded `limit` parameters allow single requests to potentially cause OOM (Out of Memory) kills by loading entire tables.

#### Phase 2: Operational Repairs (Quick-Wins)
5.  **Fix Dockerfile HEALTHCHECK (High)**
    * **Task**: Remove the `HEALTHCHECK` directive using `curl` from the Dockerfile.
    * **Reason**: `curl` does not exist in the `scratch` base image; the check will always fail. Kubernetes probes already handle this functionality.
6.  **Decouple Liveness Probe from Database (High)**
    * **Task**: Update the `/health` endpoint to return a static 200 OK. Ensure only `/readiness` checks database connectivity.
    * **Reason**: A transient DB blip currently triggers a Liveness failure, causing Kubernetes to restart the pod unnecessarily and creating "thundering herd" issues.
7.  **Implement Graceful Shutdown (High)**
    * **Task**: Add an `os.Signal` listener (SIGTERM/SIGINT) in `main()` and call `e.Shutdown(ctx)` with a timeout.
    * **Reason**: The server currently terminates immediately on SIGTERM, which can drop in-flight requests during pod rotations.

#### Phase 3: Reliability & Logic Correction
8.  **Fix Proto3 Boolean Filtering (Medium)**
    * **Task**: Modify `connect_server.go` to distinguish between a "false" value and an "unset" value for filters like `validated`.
    * **Reason**: Standard Proto3 zero-value semantics cause `validated=false` to be ignored by the filter mapping.
9.  **Stop Error Masking in Storage (Medium)**
    * **Task**: Refactor boolean check functions (e.g., `Exist`, `IsUserOwner`) to return `(bool, error)` instead of just `bool`.
    * **Reason**: Database connection drops are currently caught and returned as `false`, causing the API to return a "404 Not Found" instead of a proper "500 Internal Error".
10. **Correct Context Initialization (Medium)**
    * **Task**: Ensure `thingmodule.New` does not retain the 10-second bootstrap context for long-running processes.
    * **Reason**: Using a timeout-bound context for component initialization can cause background tasks or connection pools to die silently after 10 seconds.

#### Phase 4: Refactoring & Technical Debt
11. **Optimize DB Operations (Low)**
    * **Task**: Use PostgreSQL `RETURNING *` clauses in `Create` and `Update` methods.
    * **Reason**: Eliminates unnecessary second round-trip `Get` calls to the database.
12. **Clean Up Generated Code & Docs (Low)**
    * **Task**: Remove stale `api/thing.yaml` and rename `thing_types.gen.go` to `thing_types.go`.
    * **Reason**: These files are no longer generated by current tooling and create "dual sources of truth" confusion.
